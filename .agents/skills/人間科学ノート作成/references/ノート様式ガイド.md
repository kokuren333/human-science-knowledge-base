# ノート様式ガイド

## 文章方針

人間科学ナレッジベースのノートは、単なる箇条書きではなく、問いから理解へ進む読み物として書く。各ノートは「何を理解したいのか」を先に置き、直感的説明、定義、理論、研究、臨床または応用、限界へ進む。

専門用語は初出で英語を併記する。例: 予測処理（predictive processing）、強化学習（reinforcement learning）、作業記憶（working memory）。

## 階層構造

- 初学者: 比喩、日常例、最小限の用語。
- 学部・大学院初期: 代表理論、典型実験、基本文献。
- 研究者: 論争点、測定上の問題、数理モデル、未解決問題。
- 臨床家: 評価、診断、治療、患者理解への示唆。ただし個別助言にはしない。

## 推奨される流れ

1. 直感: 何が問題で、なぜこの概念が必要か。
2. 定義: 用語と範囲を明確にする。
3. 理論: 主要な説明枠組みを示す。
4. 研究: 実験、観察、レビュー、批判を整理する。
5. 臨床/応用: 実践にどう接続するかを慎重に述べる。
6. 限界: 何がまだ分かっていないかを明示する。

## 横断的な書き方

精神医学・神経科学・機械学習をつなぐときは、同じ言葉が分野ごとに異なる意味を持つことに注意する。たとえば「予測誤差」は、神経科学では神経応答の説明、強化学習では価値更新、臨床研究では症状モデルの仮説として使われる。ノートでは、それぞれの使い方と対応関係を分けて書く。

## 過度な断定を避ける表現

- 「〜である」だけでなく「〜と解釈されることがある」「〜を示唆する」「〜という仮説がある」を使い分ける。
- 臨床的主張では「研究で示されていること」「臨床現場での考え方」「未確立な仮説」を分ける。
- 根拠が弱い領域では、読者が確立事項と推測を混同しないようにする。
